# Supabase Triggers & Telemetry Pipeline

## Architecture

Raw telemetry from the Tesla Fleet API is ingested by a **Go server on a VPS** and inserted into the `telemetry_raw` table in Supabase. A database-level trigger (`trigger_process_telemetry`) fires the `process_telemetry()` PL/pgSQL function AFTER INSERT, which handles everything:

### What `process_telemetry()` does

1. **Updates `vehicle_status`** — parses the payload's `data` array (each item has a `key` and `value`) and sets the corresponding column:
   - `BatteryLevel`, `Odometer`, `VehicleSpeed`, `Location`, `Gear`, `InsideTemp`, `OutsideTemp`, `ChargeState`, `DoorState`, `TpmsPressure*`, etc.

2. **Trip Detection (gear-based)**:
   - When gear shifts to `D` (Drive) or `R` (Reverse) and `current_trip_id IS NULL` → INSERT new trip
   - When gear shifts to `P` (Park) and `current_trip_id IS NOT NULL` → UPDATE trip as complete, set `current_trip_id = NULL`
   - Temperature tracking: captures initial temp on start, updates running min/max/avg during trip

3. **Charging Detection (ChargeState-based)**:
   - `Charging`/`Starting` with no active session → INSERT new `charging_sessions` row
   - `Complete`/`Disconnected`/`Stopped` with active session → UPDATE session as complete
   - Classifies charger type: supercharger (DC power), home (near home coords), destination (AC)

## Key Tables

| Table | Purpose |
| --- | --- |
| `telemetry_raw` | Raw telemetry payloads from Tesla |
| `vehicle_status` | Current state of the vehicle (single row per VIN) |
| `trips` | Generated by trigger from gear changes |
| `charging_sessions` | Generated by trigger from charge state changes |

## Gotchas

- **`OutsideTemp` is sent infrequently** — only ~1 in 100 telemetry records contains it (~once per 3-5 min). Don't expect every trip to have inline temp data.
- **VIN format**: Stored as `vehicle_device.XP7YGCES0RB433079` (with prefix, not bare VIN).
- **No Go source locally**: The Go telemetry ingest server lives on the VPS only, not in this repo.
- **`telemetry-server.js` is deprecated**: Don't modify it. All trip/charging logic is in the Supabase `process_telemetry` function.
